# 日记本小程序 - 讨论确认记录

> 基于 `diary-spec.html` 规格文档，经讨论确认的细节与调整。

## 一、分期规划

### 第一期（MVP - 记录功能）

- 时光 Tab（日历 + 历史浏览 + 当天片段流）
- 记一笔（心情、正文、标签选择与参数输入）
- 片段的编辑和删除
- 标签管理（新建、删除）
- 我的 Tab（导出、设置、深色模式切换）
- 预置标签
- 云开发数据库

### 第二期（后续）

- AI 规律分析（豆包 API）
- 规律 Tab 内容
- 标签筛选 & 下钻
- 标签归档功能

## 二、确认的决定

### 数据 & 逻辑

| # | 问题 | 决定 |
|---|------|------|
| 1 | 心情 emoji 映射 | 1~5 固定：1=最差 2=较差 3=一般 4=较好 5=很好 |
| 2 | 日历翻月范围 | 不限制，无数据也可翻（支持补记久远的事） |
| 3 | 同一天片段排序 | 按创建时间排序 |
| 4 | 每天片段数 | 不限 |
| 5 | 正文长度上限 | 500 字 |
| 6 | 标签上限 | 用户最多 20 个标签，单条片段最多 5 个标签。做成配置常量，方便后续调整 |
| 7 | AI 模型 | 豆包（第二期） |
| 8 | AI 调用策略 | 手动刷新，每天限 5 次，展示上次缓存结果（第二期） |
| 17 | 片段可删除 | 支持（原规格只提了修改，补充删除能力） |

## 三、相对原规格的变更

1. **Tab 从 3 个变 4 个**：时光 / 记一笔 / 规律 / 我的
2. **标签归档暂缓**：删除逻辑简化为——有引用则提示"该标签已被使用，暂不能删除"；无引用直接删
3. **片段支持删除**：原规格只提了修改
4. **AI 分析延后**：第一期不做，规律 Tab 先放占位页面
5. **标签筛选 & 下钻延后**：随 AI 分析一起做

## 五、配置常量

以下值做成可配置常量，集中管理：

```javascript
const CONFIG = {
  MAX_TAGS_PER_USER: 20,       // 用户最多标签数
  MAX_TAGS_PER_FRAGMENT: 5,    // 单条片段最多标签数
  MAX_CONTENT_LENGTH: 500,     // 正文最大字数
  MOOD_MIN: 1,                 // 心情最低分
  MOOD_MAX: 5,                 // 心情最高分
  AI_REFRESH_LIMIT_PER_DAY: 5, // AI 每日刷新次数上限（第二期）
}
```

## 六、技术方案

- **平台**：微信小程序
- **后端**：微信云开发（CloudBase），免费额度足够个人使用
- **数据库**：云开发文档型数据库（类 MongoDB），3 个核心集合：tags、fragments、insights
- **用户隔离**：`_openid` 天然隔离，权限设为"仅创建者可读写"
- **本地缓存**：`wx.setStorage` 仅作离线兜底
- **深色模式**：跟随系统 `wx.getSystemInfo` 的 `theme` 字段，CSS 变量切换
- **部署方式**：开发阶段用「预览」扫码自用，无需正式发布
