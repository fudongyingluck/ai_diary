微信小程序 · 日常记录 + AI 规律分析
交接文档 · 供 Claude Code 实现使用 · v1.0

壹项目概述

一个微信小程序，核心是记录日常，并用 AI 分析这些日常事件的规律。两类内容混合：自由文字（简短日记）+ 结构化标签（可自定义的常见事项）。结构化标签是 AI 能找出规律的前提——纯文字无法量化，打了标签才能算出"晚睡的日子心情往往更低"这类共现规律。

三个主 Tab

时光（日历 + 历史浏览）· 记一笔（悬浮记录键，居中强调）· 规律（AI 洞察与标签分析）

叁标签体系（核心设计）

标签不是简单的"打卡/数值"两类，而是 带参数的模板。标签 = 名称 + 0~多个参数字段，每个参数有自己的类型。这样"和A冲突""和B冲突"归在同一个「冲突」标签下，A/B 只是参数值——既能一键拉出"对所有人的冲突"，也能按某个人下钻。

形态例子记录时交互
无参数（纯打卡）失眠、早起、11点后睡点一下即记
文本参数冲突 → 对象=A点亮后选/填"谁"
数值参数运动 → 时长=30 分钟点亮后就地直接输入数字
多参数冲突 → 对象=A, 强度=3填多个字段

文本参数 = 可记忆选项

文本参数（如"冲突对象"）自带一个"记忆字典"：第一次填"A"会被记住，下次点该标签时"A"作为历史选项直接可选，也能新建别人。保证同一个人不写串（不会"A""老A"分散），这样 AI 和筛选才能准确按人聚合。

标签规则（已定稿）

  • 不支持修改：名称、图标、参数定义建好即固定。（因此不存在"改名回写历史"的问题）
  • 可新增：随时建新标签；记录/编辑事件时也能当场新建并打上。
  • 删除有前提：只有没有任何片段引用的标签能删；已被使用过的标签不能删，只能归档。删除前需先查 fragments 是否引用该 tagId，有则拦截并提示归档。
  • 归档：把标签 archived 置为 true。效果：记录页不再列出该标签；历史片段照常显示/筛选/统计；可一键取消归档恢复。删除是抹掉，归档是收进抽屉。

跨夜睡眠等场景

不做任何特殊归属规则。用多个平级独立标签表达（如"11点后睡""7点前起""失眠"三个并列标签），需要时一起打；筛选时支持多选标签组合（如"晚睡 且 失眠"）。

肆事件（片段）模型

放弃"一天一条日记"，改为 一天 N 条片段（fragment）。每条片段是独立记录，自带时间、心情、标签。当天视图把多笔按时间排成片段流。

  • 时间默认只记日期，时刻字段可空。用户主动填了才存时刻。补记空白日时默认只有日期、不必选时刻。
  • 补记两种方式都要：① 记录页可把时间往前调，补昨晚忘记的；② 日历点过去的空白日，直接进去补记那天内容。
  • 已有事件可修改：改正文、心情、时间、增删该笔上的标签、改参数值；修改时也能新增标签。updatedAt 记最后修改时间。
  • 当天心情取当天所有片段心情的均值，用于日历色点与当天概览。

伍数据库设计（云开发 · 文档型）

共 3 个核心集合 + 1 个缓存集合。每条文档自带 _id、_openid（=user_id，做隔离用，无需另存）。

① tags — 标签模板

// 标签建后不可改；params 为空即纯打卡标签
{
  _id: "tag_xxx",
  _openid: "用户openid",        // 多用户隔离
  name: "冲突",
  icon: "⚡",
  params: [                      // 0~多个参数定义
    {
      key: "object",           // 参数标识
      label: "对象",           // 显示名
      type: "text",            // text | number
      unit: null,               // number 时填，如 "分钟"
      required: true,            // 必填 / 选填
      options: ["A","B","领导"]  // text 参数记忆字典，新值自动追加
    }
  ],
  archived: false,             // 归档（停用但留历史）
  sortOrder: 1,
  createdAt: Date
}

② fragments — 日记片段（核心）

// 一天多笔，每笔一条；标签连同填入的参数值冗余进来，查片段不必回查 tags
{
  _id: "frag_xxx",
  _openid: "用户openid",
  date: "2026-06-23",         // 只到天，必填
  time: "23:40",              // 可空！没填即 null
  datetime: Date,             // 排序/范围查询用；无 time 时取当天 00:00
  mood: 1,                    // 1~5，可空
  content: "脑子停不下来…",     // 正文，可空
  tags: [                       // 这一笔打的标签（含参数值）
    {
      tagId: "tag_xxx",
      name: "冲突",            // 冗余，避免回查
      icon: "⚡",
      values: { object: "A" }   // 实际填入的参数值
    },
    {
      tagId: "tag_yyy",
      name: "运动",
      values: { duration: 30 }
    }
  ],
  createdAt: Date,
  updatedAt: Date
}
冗余说明因标签不可改名，tags[] 里冗余 name/icon/values 不会产生不一致问题，查片段流时一次拿全。

③ insights — AI 洞察缓存

// AI 调用有成本，按周期+筛选缓存，避免每次进页面重算
{
  _id: "insight_xxx",
  _openid: "用户openid",
  period: "2026-06",         // 2026-W25 / 2026-06 / 2026-Q2
  scope: "month",           // week | month | quarter
  tagFilter: null,           // null=综合；"tag_xxx"=某标签专属
  stats: { ... },              // 云函数算出的统计（频次/共现），留档
  cards: [
    { type: "cooccur",
      text: "晚睡的日子，心情往往更低…",
      emphasis: ["晚睡","2.3分"] }
  ],
  generatedAt: Date
}

④ tag_values — 参数字典加速（暂不实现）

文本参数的记忆选项已存在 tags.params[].options 里，够用。此集合预留给"跨标签统计某参数值（如 A 这个人总共出现多少次）"，先不建，需要再加。

索引

  • fragments：date、datetime（日历、时间范围查询）
  • tags：sortOrder、archived
  • insights：period + scope + tagFilter（命中缓存）

陆AI 规律分析

  • 混合实现：云函数先用代码算统计（频次、共现矩阵、各标签当天均心情），把结构化数字喂给大模型，模型只负责把数字写成自然语言洞察。
  • 关联口径以"天"为单位：同一天发生的事项即算关联，不计较先后顺序（因时刻可空）。
  • 措辞只说共现、不下因果断言：说"晚睡的日子，心情往往更低"，不说"晚睡导致心情低"。这条要写进 prompt。
  • 按 周 / 月 / 季 出洞察；每条洞察带具体数字佐证。
  • 支持标签专属洞察：针对单个标签生成它的规律（出现频次、趋势、与其他标签的共现、当天均心情）。

柒按标签筛选 & 导出

按标签筛选（两层下钻）

  • 先选标签（如"冲突"），支持多选组合（"晚睡 且 失眠"）。
  • 带参数的标签可再按参数值下钻：全部 / 只看对象=A / 只看对象=B。这就是"拉出对所有人的冲突，或单看某人"。
  • 筛选结果页 = 该标签的详情：发生次数、环比趋势、当天均心情、AI 针对该标签的规律、逐次记录列表。

导出

格式用途
CSV每行一条标签记录，拉进 Excel 或喂给其他工具。首行 date,time,tag,param,value,mood
JSON含日记原文的完整备份，可迁移 / 再导入
Markdown按天整理的可读文本，适合存档（可选，优先级低）

导出可选范围：本月 / 近三月 / 全部。

捌界面设计

视觉基调：宣纸 + 淡墨 + 一点朱砂印章，克制、安静，不走糖果色 App 的路子。色板：纸 #F2EFE8、墨 #2B2925、次要 #8C857A、细线 #DAD4C8、朱砂强调 #A4543B（只用于印章、AI 标签条、负向数据）、正向信号青 #6E7A66。字体偏宋体衬线，传达日记的"留笔"感。

拾留给实现者的提示

  • 建标签是"建后锁定"，UI 上要对修改入口做禁用 + 说明，避免用户期待落空。
  • 删除标签前必做引用校验（查 fragments.tags.tagId），有引用就引导归档。
  • 导出受微信小程序文件能力限制，走 wx.saveFileToDisk 或分享流。
  • AI 洞察务必先缓存（insights 集合），按 period+scope+tagFilter 命中，不每次重算。
  • AI prompt 需显式约束：以"天"为关联单位、只描述共现、禁止因果断言、每条结论附数字。
  • 视觉令牌（色板/字体）见第捌节，保持克制安静的基调。
一处待你最终确认有历史的标签删除策略，本文档按"不可删、只能归档"撰写。若希望支持强删（连带清历史引用），请告知后修订。
